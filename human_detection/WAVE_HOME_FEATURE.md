# 挥手动作功能说明

## 功能概述

**检测到人 → 执行挥手 → 回到初始位姿 [0,0,0,0,0,0]**

## 完整动作流程

```
检测到人 (距离 <= 2米)
    ↓
触发挥手动作
    ↓
1. 连接机械臂
2. 回到初始位姿 [0,0,0,0,0,0]
3. 移动到准备姿势
4. 挥手 (左右摆动3次)
5. 回到初始位姿 [0,0,0,0,0,0] ← 必须执行
6. 断开连接
    ↓
动作完成
```

## 初始位姿说明

**初始位姿**: `[0, 0, 0, 0, 0, 0]` (J1-J6 全部归零)

- J1: 0° (底座旋转)
- J2: 0° (大臂)
- J3: 0° (小臂)
- J4: 0° (手腕旋转)
- J5: 0° (手腕摆动)
- J6: 0° (手爪旋转)

这是机械臂的默认待机姿势，所有关节都回到零位。

## 安全措施

### 1. 必须回到初始位姿
无论挥手动作是否成功，最后都会执行步骤5回到初始位姿。

### 2. 异常恢复
- 如果执行过程中出错，`finally` 块会确保回到初始位姿
- 如果进程被中断，`atexit` 会执行紧急恢复
- 如果段错误，独立进程崩溃不会卡住主程序

### 3. 触发间隔保护
默认60秒内不会重复触发，防止频繁动作。

## 使用方式


当有人进入摄像头视野，且距离在2米范围内时：
1. 系统自动检测到人
2. 触发挥手动作
3. 机械臂执行完整流程
4. 回到初始位姿 [0,0,0,0,0,0]

### 手动触发
在 Web GUI 的"挥手动作"面板中点击"触发挥手"按钮。

## 配置修改

编辑 `web_gui.py` 中的配置：

```python
config = {
    "enable_wave_action": True,   # 启用挥手
    "wave_interval": 60,          # 触发间隔(秒)
    "welcome_distance": 2.0,      # 迎宾距离(米)
}
```

## 故障排除

### 问题: 挥手后没有回到初始位姿
**解决**: 检查机械臂连接，确保步骤5执行完成

### 问题: 无法触发
**检查**:
1. 确认 `enable_wave_action: True`
2. 检查触发间隔 (`wave_interval`)
3. 检查迎宾距离 (`welcome_distance`)

### 问题: 机械臂不动作
**测试**:
```bash
python3 src/wave_hand_standalone.py
```
如果独立脚本可以执行，说明 Web 端配置有问题。
